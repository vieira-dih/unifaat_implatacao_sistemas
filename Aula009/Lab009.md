# Lab 009 - Computação com EC2 e Segurança de Rede

## Objetivos
- Provisionar e configurar instâncias EC2
- Implementar Security Groups para controle de acesso
- Configurar chaves SSH para acesso seguro
- Criar e gerenciar VPC com subnets públicas e privadas
- Migrar aplicação local para EC2
- Implementar boas práticas de segurança de rede

## Pré-requisitos
- AWS CLI configurado (Aula 008)
- Conta AWS com Free Tier ativo
- Conhecimento básico de Linux
- Aplicação containerizada das aulas anteriores

## Parte 1: Configuração da VPC e Rede (30 min)

### 1.1 Criação da VPC

```bash
# Criar VPC
aws ec2 create-vpc \
    --cidr-block 10.0.0.0/16 \
    --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=Lab009-VPC}]'

# Anotar o VPC ID retornado
export VPC_ID=vpc-xxxxxxxxx

# Habilitar DNS hostnames
aws ec2 modify-vpc-attribute \
    --vpc-id $VPC_ID \
    --enable-dns-hostnames
```

### 1.2 Criação das Subnets

```bash
# Subnet Pública (para Web Server)
aws ec2 create-subnet \
    --vpc-id $VPC_ID \
    --cidr-block 10.0.1.0/24 \
    --availability-zone us-east-1a \
    --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=Lab009-Public-Subnet}]'

export PUBLIC_SUBNET_ID=subnet-xxxxxxxxx

# Subnet Privada (para Database)
aws ec2 create-subnet \
    --vpc-id $VPC_ID \
    --cidr-block 10.0.2.0/24 \
    --availability-zone us-east-1a \
    --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=Lab009-Private-Subnet}]'

export PRIVATE_SUBNET_ID=subnet-xxxxxxxxx

# Configurar subnet pública para auto-assign IP público
aws ec2 modify-subnet-attribute \
    --subnet-id $PUBLIC_SUBNET_ID \
    --map-public-ip-on-launch
```

### 1.3 Configuração do Internet Gateway

```bash
# Criar Internet Gateway
aws ec2 create-internet-gateway \
    --tag-specifications 'ResourceType=internet-gateway,Tags=[{Key=Name,Value=Lab009-IGW}]'

export IGW_ID=igw-xxxxxxxxx

# Anexar IGW à VPC
aws ec2 attach-internet-gateway \
    --internet-gateway-id $IGW_ID \
    --vpc-id $VPC_ID
```

### 1.4 Configuração das Route Tables

```bash
# Obter Route Table padrão da VPC
export ROUTE_TABLE_ID=$(aws ec2 describe-route-tables \
    --filters "Name=vpc-id,Values=$VPC_ID" \
    --query 'RouteTables[0].RouteTableId' \
    --output text)

# Adicionar rota para internet na subnet pública
aws ec2 create-route \
    --route-table-id $ROUTE_TABLE_ID \
    --destination-cidr-block 0.0.0.0/0 \
    --gateway-id $IGW_ID

# Associar route table à subnet pública
aws ec2 associate-route-table \
    --route-table-id $ROUTE_TABLE_ID \
    --subnet-id $PUBLIC_SUBNET_ID
```

**✅ Checkpoint 1:** VPC com subnets pública e privada criada

## Parte 2: Configuração de Segurança (20 min)

### 2.1 Criação de Key Pair

```bash
# Criar key pair para acesso SSH
aws ec2 create-key-pair \
    --key-name Lab009-KeyPair \
    --query 'KeyMaterial' \
    --output text > Lab009-KeyPair.pem

# Configurar permissões corretas
chmod 400 Lab009-KeyPair.pem

# Verificar criação
aws ec2 describe-key-pairs --key-names Lab009-KeyPair
```

### 2.2 Security Groups

#### Security Group para Web Server
```bash
# Criar Security Group para Web Server
aws ec2 create-security-group \
    --group-name Lab009-WebServer-SG \
    --description "Security Group para Web Server" \
    --vpc-id $VPC_ID \
    --tag-specifications 'ResourceType=security-group,Tags=[{Key=Name,Value=Lab009-WebServer-SG}]'

export WEB_SG_ID=sg-xxxxxxxxx

# Regras de entrada
# SSH (apenas do seu IP)
MY_IP=$(curl -s https://checkip.amazonaws.com)
aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG_ID \
    --protocol tcp \
    --port 22 \
    --cidr $MY_IP/32

# HTTP
aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG_ID \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0

# HTTPS
aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG_ID \
    --protocol tcp \
    --port 443 \
    --cidr 0.0.0.0/0

# Porta customizada da aplicação
aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG_ID \
    --protocol tcp \
    --port 3000 \
    --cidr 0.0.0.0/0
```

#### Security Group para Database
```bash
# Criar Security Group para Database
aws ec2 create-security-group \
    --group-name Lab009-Database-SG \
    --description "Security Group para Database" \
    --vpc-id $VPC_ID \
    --tag-specifications 'ResourceType=security-group,Tags=[{Key=Name,Value=Lab009-Database-SG}]'

export DB_SG_ID=sg-xxxxxxxxx

# Permitir acesso MySQL apenas do Web Server SG
aws ec2 authorize-security-group-ingress \
    --group-id $DB_SG_ID \
    --protocol tcp \
    --port 3306 \
    --source-group $WEB_SG_ID

# Permitir SSH apenas do Web Server SG (para administração)
aws ec2 authorize-security-group-ingress \
    --group-id $DB_SG_ID \
    --protocol tcp \
    --port 22 \
    --source-group $WEB_SG_ID
```

**✅ Checkpoint 2:** Security Groups configurados com princípio do menor privilégio

## Parte 3: Provisionamento de Instâncias (40 min)

### 3.1 Instância Web Server

#### User Data Script para Web Server
Crie `web-server-userdata.sh`:
```bash
#!/bin/bash
yum update -y
yum install -y docker git

# Iniciar Docker
systemctl start docker
systemctl enable docker
usermod -a -G docker ec2-user

# Instalar Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# Criar diretório para aplicação
mkdir -p /home/ec2-user/app
chown ec2-user:ec2-user /home/ec2-user/app

# Log de inicialização
echo "Web Server setup completed at $(date)" > /var/log/user-data.log
```

#### Lançar Instância Web Server
```bash
# Obter AMI ID do Amazon Linux 2
AMI_ID=$(aws ec2 describe-images \
    --owners amazon \
    --filters "Name=name,Values=amzn2-ami-hvm-*-x86_64-gp2" \
    --query 'Images | sort_by(@, &CreationDate) | [-1].ImageId' \
    --output text)

# Lançar instância
aws ec2 run-instances \
    --image-id $AMI_ID \
    --count 1 \
    --instance-type t3.micro \
    --key-name Lab009-KeyPair \
    --security-group-ids $WEB_SG_ID \
    --subnet-id $PUBLIC_SUBNET_ID \
    --user-data file://web-server-userdata.sh \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Lab009-WebServer}]'

# Anotar Instance ID
export WEB_INSTANCE_ID=i-xxxxxxxxx

# Aguardar instância ficar running
aws ec2 wait instance-running --instance-ids $WEB_INSTANCE_ID

# Obter IP público
export WEB_PUBLIC_IP=$(aws ec2 describe-instances \
    --instance-ids $WEB_INSTANCE_ID \
    --query 'Reservations[0].Instances[0].PublicIpAddress' \
    --output text)

echo "Web Server IP: $WEB_PUBLIC_IP"
```

### 3.2 Instância Database Server

#### User Data Script para Database
Crie `db-server-userdata.sh`:
```bash
#!/bin/bash
yum update -y
yum install -y mysql-server

# Configurar MySQL
systemctl start mysqld
systemctl enable mysqld

# Configuração básica do MySQL
mysql -e "CREATE DATABASE IF NOT EXISTS appdb;"
mysql -e "CREATE USER 'appuser'@'%' IDENTIFIED BY 'SecurePass123!';"
mysql -e "GRANT ALL PRIVILEGES ON appdb.* TO 'appuser'@'%';"
mysql -e "FLUSH PRIVILEGES;"

# Configurar MySQL para aceitar conexões externas
sed -i 's/bind-address.*/bind-address = 0.0.0.0/' /etc/mysql/mysql.conf.d/mysqld.cnf
systemctl restart mysqld

echo "Database Server setup completed at $(date)" > /var/log/user-data.log
```

#### Lançar Instância Database
```bash
# Lançar instância de banco
aws ec2 run-instances \
    --image-id $AMI_ID \
    --count 1 \
    --instance-type t3.micro \
    --key-name Lab009-KeyPair \
    --security-group-ids $DB_SG_ID \
    --subnet-id $PRIVATE_SUBNET_ID \
    --user-data file://db-server-userdata.sh \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Lab009-Database}]'

export DB_INSTANCE_ID=i-xxxxxxxxx

# Aguardar instância ficar running
aws ec2 wait instance-running --instance-ids $DB_INSTANCE_ID

# Obter IP privado
export DB_PRIVATE_IP=$(aws ec2 describe-instances \
    --instance-ids $DB_INSTANCE_ID \
    --query 'Reservations[0].Instances[0].PrivateIpAddress' \
    --output text)

echo "Database Server Private IP: $DB_PRIVATE_IP"
```

### 4.2 Preparar Aplicação na Instância

```bash
# Na instância EC2, criar estrutura da aplicação
mkdir -p ~/app/{frontend,backend}
cd ~/app

# Criar aplicação backend simples
cat > backend/package.json << 'EOF'
{
  "name": "lab009-backend",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "express": "^4.18.0",
    "mysql2": "^3.6.0"
  }
}
EOF

# Criar servidor backend
cat > backend/server.js << 'EOF'
const express = require('express');
const mysql = require('mysql2/promise');
const app = express();
const PORT = process.env.PORT || 3000;

app.use(express.json());
app.use(express.static('public'));

// Database connection
const dbConfig = {
  host: process.env.DB_HOST || 'localhost',
  user: process.env.DB_USER || 'appuser',
  password: process.env.DB_PASSWORD || 'SecurePass123!',
  database: process.env.DB_NAME || 'appdb'
};

// Health check
app.get('/health', async (req, res) => {
  try {
    const connection = await mysql.createConnection(dbConfig);
    await connection.ping();
    await connection.end();
    res.json({ 
      status: 'healthy', 
      timestamp: new Date().toISOString(),
      database: 'connected',
      server: 'EC2'
    });
  } catch (error) {
    res.status(503).json({ 
      status: 'unhealthy', 
      error: error.message 
    });
  }
});

// API endpoints
app.get('/api/info', (req, res) => {
  res.json({
    message: 'API rodando no EC2!',
    instance: process.env.INSTANCE_ID || 'unknown',
    region: process.env.AWS_REGION || 'us-east-1'
  });
});

app.listen(PORT, '0.0.0.0', () => {
  console.log(`Server running on port ${PORT}`);
});
EOF

# Criar Dockerfile
cat > backend/Dockerfile << 'EOF'
FROM node:16-alpine

RUN apk add --no-cache curl

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

HEALTHCHECK --interval=30s --timeout=3s --start-period=10s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

EXPOSE 3000
CMD ["npm", "start"]
EOF
```

### 4.4 Configurar Docker Compose

```bash
# Criar arquivo de ambiente
cat > .env << EOF
DB_HOST=$DB_PRIVATE_IP
DB_USER=appuser
DB_PASSWORD=SecurePass123!
DB_NAME=appdb
PORT=3000
INSTANCE_ID=$(curl -s http://169.254.169.254/latest/meta-data/instance-id)
AWS_REGION=$(curl -s http://169.254.169.254/latest/meta-data/placement/region)
EOF

# Criar docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  app:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DB_HOST=${DB_HOST}
      - DB_USER=${DB_USER}
      - DB_PASSWORD=${DB_PASSWORD}
      - DB_NAME=${DB_NAME}
      - PORT=${PORT}
      - INSTANCE_ID=${INSTANCE_ID}
      - AWS_REGION=${AWS_REGION}
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./frontend:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - app
    restart: unless-stopped
EOF

# Criar configuração do Nginx
cat > nginx.conf << 'EOF'
events {
    worker_connections 1024;
}

http {
    upstream backend {
        server app:3000;
    }
    
    server {
        listen 80;
        server_name _;
        
        # Servir frontend
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ /index.html;
        }
        
        # Proxy para API
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
        
        # Health check
        location /health {
            proxy_pass http://backend/health;
        }
    }
}
EOF
```

## Parte 6: Limpeza de Recursos (10 min)

### 6.1 Terminar Instâncias

```bash
# Terminar instâncias
aws ec2 terminate-instances --instance-ids $WEB_INSTANCE_ID $DB_INSTANCE_ID

# Aguardar terminação
aws ec2 wait instance-terminated --instance-ids $WEB_INSTANCE_ID $DB_INSTANCE_ID
```

### 6.2 Limpar Recursos de Rede

```bash
# Deletar Security Groups
aws ec2 delete-security-group --group-id $WEB_SG_ID
aws ec2 delete-security-group --group-id $DB_SG_ID

# Desanexar e deletar Internet Gateway
aws ec2 detach-internet-gateway --internet-gateway-id $IGW_ID --vpc-id $VPC_ID
aws ec2 delete-internet-gateway --internet-gateway-id $IGW_ID

# Deletar Subnets
aws ec2 delete-subnet --subnet-id $PUBLIC_SUBNET_ID
aws ec2 delete-subnet --subnet-id $PRIVATE_SUBNET_ID

# Deletar VPC
aws ec2 delete-vpc --vpc-id $VPC_ID

# Deletar Key Pair
aws ec2 delete-key-pair --key-name Lab009-KeyPair
rm -f Lab009-KeyPair.pem
```

## Comandos de Troubleshooting

### Problemas Comuns

#### Instância não inicia
```bash
# Verificar status da instância
aws ec2 describe-instances --instance-ids $INSTANCE_ID

# Verificar logs de inicialização
aws ec2 get-console-output --instance-id $INSTANCE_ID
```

#### Não consegue conectar via SSH
```bash
# Verificar Security Group
aws ec2 describe-security-groups --group-ids $WEB_SG_ID

# Verificar se instância tem IP público
aws ec2 describe-instances --instance-ids $WEB_INSTANCE_ID \
    --query 'Reservations[0].Instances[0].PublicIpAddress'

# Testar conectividade
telnet $WEB_PUBLIC_IP 22
```

#### Aplicação não conecta ao banco
```bash
# Verificar conectividade de rede
# Na instância web:
telnet $DB_PRIVATE_IP 3306

# Verificar Security Group do banco
aws ec2 describe-security-groups --group-ids $DB_SG_ID

# Verificar logs da aplicação
sudo docker-compose logs app
```
