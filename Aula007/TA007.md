## 1. Conceitos Fundamentais

### O que é uma Revisão Prática Individual?

Uma **revisão prática individual** é uma sessão focada em consolidar e verificar o domínio dos conceitos aprendidos através da prática hands-on. É o momento de "afinar os instrumentos" antes da apresentação principal - a prova.

#### Objetivos da Revisão:
- **Consolidar conhecimentos** das aulas 1-6
- **Identificar lacunas** no aprendizado
- **Praticar troubleshooting** em ambiente controlado
- **Ganhar confiança** para a prova
- **Esclarecer dúvidas** finais

### Metodologia de Revisão Eficaz

#### 1. **Revisão Ativa vs Passiva**

**Revisão Passiva (❌ Menos eficaz):**
- Apenas reler anotações
- Assistir vídeos sem praticar
- Memorizar comandos sem contexto

**Revisão Ativa (✅ Mais eficaz):**
- Implementar projetos do zero
- Resolver problemas simulados
- Explicar conceitos para colegas
- Criar documentação própria

#### 2. **Técnica de Revisão Espiral**
Em vez de revisar linearmente (aula 1 → 2 → 3...), use a abordagem espiral:
- **1ª volta**: Conceitos principais de cada aula
- **2ª volta**: Conexões entre as aulas
- **3ª volta**: Aplicação prática integrada
- **4ª volta**: Troubleshooting e edge cases

### Padrões de Problemas Comuns

#### **Categoria 1: Problemas de Configuração**
```bash
# Sintomas
- Container não inicia
- Erro de sintaxe no YAML
- Variáveis de ambiente não carregam

# Diagnóstico
docker-compose config
docker-compose logs service-name

# Soluções típicas
- Verificar indentação YAML
- Validar arquivo .env
- Checar sintaxe do Dockerfile
```

#### **Categoria 2: Problemas de Conectividade**
```bash
# Sintomas
- App não conecta ao banco
- Nginx retorna 502 Bad Gateway
- Healthcheck sempre failing

# Diagnóstico
docker-compose exec app ping db
docker-compose logs nginx
curl -I http://localhost/health

# Soluções típicas
- Verificar nomes dos serviços
- Confirmar portas corretas
- Aguardar inicialização completa
```

#### **Categoria 3: Problemas de Persistência**
```bash
# Sintomas
- Dados perdidos após restart
- Banco não inicializa
- Volumes não montam

# Diagnóstico
docker volume ls
docker-compose exec db ls -la /var/lib/mysql

# Soluções típicas
- Configurar volumes corretamente
- Verificar permissões de arquivo
- Aguardar inicialização do banco
```

### Estratégias de Troubleshooting

#### **Método PDCA para Debug:**

##### **P (Plan) - Planejar**
- Identificar o problema específico
- Formular hipóteses sobre a causa
- Definir testes para validar hipóteses

##### **D (Do) - Executar**
- Implementar uma solução por vez
- Documentar cada tentativa
- Manter logs das mudanças

##### **C (Check) - Verificar**
- Testar se o problema foi resolvido
- Validar que não criou novos problemas
- Confirmar funcionamento completo

##### **A (Act) - Agir**
- Documentar a solução encontrada
- Aplicar aprendizado a problemas similares
- Compartilhar conhecimento com equipe

#### **Comandos de Diagnóstico Essenciais:**
```bash
# Status geral
docker-compose ps
docker stats --no-stream

# Logs detalhados
docker-compose logs --tail=50 service-name
docker-compose logs -f

# Conectividade
docker-compose exec service ping target
curl -v http://localhost:port/endpoint

# Recursos
df -h  # Espaço em disco
free -h  # Memória
netstat -tlnp  # Portas em uso
```

## 2. Questões de Múltipla Escolha

### Questão 1
Qual é a abordagem mais eficaz para revisar conceitos técnicos antes de uma prova prática?

a) Ler todas as anotações das aulas anteriores
b) Assistir vídeos sobre as tecnologias
c) Implementar um projeto completo do zero
d) Memorizar todos os comandos Docker

**Gabarito: C**

**Justificativa:** A revisão ativa através da implementação prática é mais eficaz que métodos passivos. Implementar um projeto do zero força você a aplicar todos os conceitos de forma integrada, identificando lacunas no conhecimento e ganhando confiança prática.

### Questão 2
Durante uma revisão prática, você encontra um erro que não consegue resolver rapidamente. Qual é a melhor estratégia?

a) Desistir e partir para outro tópico
b) Passar horas tentando resolver sem método
c) Aplicar método sistemático de troubleshooting e documentar tentativas
d) Pedir ajuda imediatamente

**Gabarito: C**

**Justificativa:** O método sistemático de troubleshooting (identificar, diagnosticar, testar, documentar) é mais eficiente que tentativas aleatórias. Documentar as tentativas evita repetir erros e ajuda a identificar padrões, sendo uma habilidade valiosa para a prova.

### Questão 3
Qual é o principal objetivo de uma revisão prática individual?

a) Memorizar todos os comandos possíveis
b) Consolidar conhecimentos e identificar lacunas antes da avaliação
c) Comparar desempenho com outros alunos
d) Criar um projeto mais complexo que os anteriores

**Gabarito: B**

**Justificativa:** O objetivo principal da revisão é consolidar o aprendizado e identificar pontos que precisam de reforço antes da avaliação formal. É um momento de autoavaliação e preparação, não de competição ou complexidade adicional.
