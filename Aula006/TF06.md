# TF06 - Configuração do Ambiente AWS e Primeiros Passos com a CLI
**Disciplina:** Implementação de Sistemas  
**Curso:** Análise e Desenvolvimento de Sistemas - UniFAAT  
**Aula:** 06 - Preparação do Ambiente para a Cloud  
**Valor:** 1.5 pontos  
**Prazo:** 1 semana após a aula

## Objetivo

Familiarizar-se com o ambiente AWS, configurar o acesso programático de forma segura através de um usuário IAM e executar os primeiros comandos de interação com os serviços da AWS via Command-Line Interface (CLI).

## Tarefa

A tarefa consiste em 3 partes principais:
1.  **Segurança de Faturamento:** Criar um alerta de faturamento para monitorar seus custos.
2.  **Gerenciamento de Acesso:** Criar um usuário IAM dedicado para acesso programático com permissões mínimas.
3.  **Configuração Local:** Instalar e configurar a AWS CLI na sua máquina local para interagir com a sua conta AWS.

## Especificações e Requisitos Técnicos

### Parte 1: Alerta de Faturamento (Billing Alert)

O primeiro passo antes de começar a usar a AWS é garantir que você não terá surpresas na fatura.

- [ ] **(0.25 pts)** Acessar o console de Faturamento (Billing) da AWS.
- [ ] **(0.25 pts)** Criar um **alerta de faturamento** que o notifique por e-mail se os custos estimados excederem **$5.00 (cinco dólares)**.
- [ ] **Evidência:** Um screenshot da tela de confirmação do alerta criado ou da lista de alertas mostrando o que foi configurado.

### Parte 2: Criação do Usuário IAM

Nunca use seu usuário "root". Para acesso via CLI, crie um usuário com permissões restritas.

- [ ] **(0.25 pts)** Acessar o serviço **IAM** (Identity and Access Management).
- [ ] Criar um novo usuário com um nome claro (ex: `cli-user`).
    - [ ] Habilitar **"Access key - Programmatic access"**.
- [ ] Anexar as seguintes políticas de permissão **gerenciadas pela AWS**:
    - [ ] **(0.25 pts)** `AmazonS3ReadOnlyAccess`
    - [ ] **(0.25 pts)** `AmazonEC2ReadOnlyAccess`
- [ ] **(Não pontuado, mas crucial)** Salvar a **Access Key ID** e a **Secret Access Key** em um local seguro na sua máquina. **Esta é a única vez que a Secret Key será exibida.**
- [ ] **Evidência:** Um screenshot da tela de resumo do usuário, mostrando o usuário criado e as políticas de permissão anexadas.

### Parte 3: Instalação e Uso da AWS CLI

Agora, vamos configurar sua máquina para "conversar" com a AWS.

- [ ] **(Não pontuado, mas necessário)** Fazer o download e instalar a **AWS CLI v2** para o seu sistema operacional.
- [ ] Abrir seu terminal (Prompt de Comando, PowerShell, ou Terminal do Linux/macOS).
- [ ] Executar o comando `aws configure` e preencher com:
    - A **Access Key ID** e **Secret Access Key** do usuário IAM que você criou.
    - `us-east-1` como **Default region name**.
    - `json` como **Default output format**.
- [ ] **(0.25 pts)** Executar os seguintes comandos e verificar se a saída é bem-sucedida (sem erros de acesso):
    - `aws s3 ls` (Deve listar os buckets S3 - pode estar vazio).
    - `aws ec2 describe-regions` (Deve listar as regiões da AWS).
- [ ] **Evidência:** Um screenshot do seu terminal mostrando a execução do `aws configure` (oculte suas chaves!) e a saída bem-sucedida dos comandos `aws s3 ls` e `aws ec2 describe-regions`.

## Dicas

1.  **Segurança em Primeiro Lugar:** **NUNCA** compartilhe ou faça commit das suas chaves de acesso (`Access Key` e `Secret Key`). Trate-as como senhas.
2.  **Princípio do Menor Privilégio:** Sempre conceda apenas as permissões estritamente necessárias. Note que demos apenas acesso de "leitura" (`ReadOnly`).
3.  **Monitore seus Custos:** O alerta de faturamento é seu melhor amigo para evitar surpresas. Verifique o painel de custos regularmente.
